# 数据分析要求：

关系型数据库：PostgressSQL、MySQL等，熟悉（增删改查操作）

# AIGC：基于大模型生成内容开发

1. 低代码平台（低性能）：Dify
2. LangChain、LangGraph（高性能）

LangChain：构建链式工作流，支持多个大模型协作分工。

通用LLM的知识具有局限性 -> LangChain RAG模块：构建本地知识对大模型进行检索增强生成。数据需要切块（向量数据库），整合成结构化数据供模型学习理解。流程为：数据->加载 ->转换->向量化嵌入->存储->检索->LLM。检索的要求：速度快、检索出来相似度最高的内容语义信息也要相关。

LangGraph：多智能体协作开发。

# 建模重要性分配

数据质量：收集、清洗、数据增强最为重要。

建模：模型选择最不重要。
